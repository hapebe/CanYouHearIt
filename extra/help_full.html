Quick Start Instructions

First of all, have you already run the program? For a short function check, just click the icon in the top-left corner of this window. If that does work, you're lucky - and I'm surprised. If you get an error message, you'll have to download and install the Java Runtime Environment 1.3 . See Requirements for further help.

When the program is ready, you will see it's Surface. If you get a "Midi not available" error, see Requirements

  • Choose Series of Intervals or Series of Chords from the user menu. You are now in a lesson!
  • Now Press the Play Button You hear the sound you should analyze. If you don't hear anything, see FAQ.
  • Choose the base tone and the type of the sound. Just click on one of the buttons.
  • If you're sure you can't guess any better, press Guess.
  • You get an evaluation of your trial now, then it's up for the next task. Again Play and so on...
  • Your current progress in the lesson is shown below the Play and Stop button. One Lesson consists of 50 exercises. If you're finished with a lesson, you get a more comprehensive evaluation screen, and then you're free to start something new. Just in case you don't want to fulfil all these tasks just now, or anyway at anytime, you can activate Save&Exit from the File Menu. It's then, that you're shown a message that should make you read on at Applet/Application.
That's all you absolutely need to know.

Train interval recognition

Choose User / Train Intervals to start a lesson of this type.

This lesson type create randomly chosen intervals between a minor second and an octave. The base tone can vary from C1 to C5, i.e. the higher, second tone can be C#1 to C6. The two notes are played as a chord, when you press the play button. You should specify the base tone and the interval (defining the second note, too.) by pressing the appropriate buttons. You can omit these clicks, but that will be taken as wrong. If you're sure you can't do better, press Guess. After the evaluation message, the lesson continues with the next interval.

In the type button field there are some abbreviations:

  • < means a minor interval ( "<2" means minor second )
  • > means a major interval
  • tt stands for Tritonus (the interval that consists of three whole notes)
If you are worried about your success rate concerning base tones: note on absolute hearing

Train chord recognition

Choose User / Train Chords to start a lesson of this type.

This lesson type creates randomly chosen chords of certain types (see Abbreviation Table). The base tone can vary from C1 to C5, the additional notes are all higher in the current version. The chord sounds when you press the play button. You should specify the base tone and the chord type by pressing the appropriate buttons. You can omit these clicks, but that will be taken as wrong. If you're sure you can't do better, press Guess. After the evaluation message, the lesson continues with the next chord.

Statistics

"Can You Hear It?" records your success and failure...
While practising a type of lesson, you see a colorful section on the right. In general, green means good and red means bad. The stats are assorted by the type of sound and additionally, for the every single base tone and type. If you move your mouse over a single section of the stats, a tooltip appears telling you your overall performance on that special item.

Some notes on the statistics:

  • Stats can only be saved from session to session in Application mode! (Applet/Application)
  • Stats are automatically loaded on every user login.
  • Stats are automatically saved on exit or switch user.
  • The length of the color bars increases with the number of trials on the concerned sound.
  • Stats are only modified after a guess. Starting a new lesson or quitting etc. doesn't affect the stats.
  • Don't panic if you see nothing: Stats are only visible during a lesson.

User Management

This feature applies only to the Application Mode. (Applet/Application) "Can You Hear It?" supports more than one user at one machine.

  • Every user has his or her own stats record.
That makes it easier for everyone to keep track of one's progress. For the moment, the different user accounts are not secured by a password or such a thing. Probably I'll include that in future versions.
The usage is very simple, indeed:
When you launch "Can You Hear It?" as an Application, you're immediately prompted for your user name.
  • If, right now, you have never chosen one before: just type in any name you want. That's your user name now.
The current user is now showing in the menu bar (User:xxxx...). All your efforts are now recorded personally for you. The next time you "log in" with the same user name, your statistics are available again. If you want to change the current user from the running program, Switch User... in the User menu.

Applet? Application?

"Can You Hear It?" is a Java program. Java provides two ways to run programs on a desktop computer: "Applets" and "Applications". Applets are more popular; they are supported by the major internet browsers. That means, they run within common HTML-Pages. Applications are (almost) stand-alone programs like any other software on your computer. "Can You Hear It?" runs both as an applet and as an application. If you run it in application mode, you profit of the application's ability to read and write files. The applet version cannot save&load stats, because applets are not allowed to access the file system of your computer.
To run "Can You Hear It?" as an application, you have to

  1. Add the Java Interpreter to your command path. (Windows users: add path %path%;c:\jre1.3\bin\ to your c:\autoexec.bat ,where c:\jre1.3\bin\ must be replaced ba the directory containing java.exe . You have to restart your computer, then.)
  2. Open a shell or command prompt.
  3. Change directory to the "Can You Hear It?" directory (where you installed it to).
  4. Type in: java Music<ENTER>
Instead of step 2-4, Windows users simply can double-click cyhi.bat provided in the program directory.
If you can't start "Can You Hear It?" at all, read on at Requirements.

Requirements

Going insane about getting cryptic error messages? Class not found? Well: "Can You Hear It?" 1.0 is quite fastidious.
You need:

  • Java 2 Platform Runtime Environment Version 1.3 including Java Plug-In for your computer system
  • A soundcard with a MIDI synthesizer. With a Non-Windows operation system you should not even need a MIDI capable sound card, but that's something I don't know. Those guys at Sun say so.
The capabilities of your standard browser are not sufficient for this program! "Can You Hear It!" uses progressive sound technologies, which aren't available with older Java versions supported by Netscape Navigator or MS Internet Explorer.
You won't even succeed with a Java 1.2 version on your computer!

Frequently Asked Questions

I don't hear any sound at all!

Are you sure you did not get an error message such as "Couldn't open MIDI Sythesizer"?
If that is the case, either there is another programm using the MIDI-Syntesizer (maybe another instance of the program in another browser window?) or there's a general problem with MIDI synthesis. You can check your soundcard configuration and retry.
If there is no error message but no sound, either: All I can advise you is to check your Speakers/Headphones and their plugs and your Mixer Software. Probably MIDI synthesis is deactivated there...?
Sorry, if that didn't help...

Can I improve the quality of sound?

Maybe. If you use a Non-Windows System and haven't yet downloaded the "Java Soundbank - deluxe" you could do that.

  • http://java.sun.com/products/java-media/sound/
Download the deluxe soundbanks and follow the Install Directions. The next time you run "Can You Hear It?", you should be able to hear it...

Why doesn't the "Mac" Look&Feel work?

I don't know; I use the official way to establish it. Ask Sun...

How am I supposes to guess base tones?

Indeed, few people can tell the absolute pitch of some tone as they hear it. However, maybe you are one of them?
As you become more experienced, you will find it useful to determine the current base tone remembering the previous one... Finally you don't have to worry if you fail most of the time - that's just normal amongst amateurs.
